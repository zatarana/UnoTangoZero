## Convenção de Versionamento

- **MAJOR**: Breaking changes na API ou funcionalidade
- **MINOR**: Novas features (backward compatible)
- **PATCH**: Bug fixes e melhorias pequenas

Exemplo: `v1.2.3` significa:
- v1 = major version
- 2 = minor version (5 novas features nesta release)
- 3 = patch version (3 bug fixes nesta release)
